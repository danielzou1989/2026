# Bitget 适配字段映射与接口说明

本文件用于实现 Bitget 适配器时的字段映射与接口说明。字段名与端点以 Bitget 官方文档为准，本文件提供统一接口的映射逻辑与常见字段语义。

## 符号与市场规范化

- 统一 `symbol` 输入：策略层接受 `BTCUSDT` 或 `BTC/USDT`，适配器内部转换为 Bitget 所需格式
- 合约市场通常需要 `productType` 与 `marginCoin`，与 `symbol` 一起作为路由条件
- 现货与合约接口不同，适配器需根据 `market_type`（spot/mix）切换

## get_account_info()

### 目标标准字段

- `asset`, `total`, `available`, `used`, `risk_rate`

### Bitget 映射建议

- 现货资产：从资产列表中读取 `available` 与 `frozen`（或 `locked`），`total = available + frozen`
- 合约账户：优先使用账户权益类字段（如 `equity`/`availableEquity`/`margin`），`used = total - available`
- `risk_rate`：若接口提供 `riskRate` 或类似字段直接映射；否则返回 `null`

### 说明

若字段名不一致，按语义选择最接近字段；若无法计算则返回 `null` 并在 `error` 中说明。

## get_positions()

### 目标标准字段

- `symbol`, `side`, `qty`, `entry_price`, `mark_price`, `upl`, `upl_pct`

### Bitget 映射建议

- `qty`：使用持仓数量字段（如 `total`, `available`, `holdVol` 等），优先总持仓
- `entry_price`：平均开仓价字段（如 `averagePrice`/`openPriceAvg`）
- `mark_price`：若提供 `markPrice` 映射；否则使用 `last` 或 `price` 并标记为近似
- `upl`：未实现盈亏字段（如 `unrealizedPL`/`upl`）
- `upl_pct`：若返回 `uplRate`/`profitRate` 直接映射，否则用 `upl / (entry_price * qty)` 近似
- `side`：按 Bitget 返回的多空标识映射为 `buy`/`sell`

## place_order(symbol, side, type, qty, price?, reduce_only?, leverage?)

### 统一参数到 Bitget 参数

- `side`: `buy`/`sell`
- `type`: `market`/`limit`
- `qty`: 对应 `size` 或 `quantity`
- `price`: 限价单必传
- `reduce_only`: 合约场景映射为 `reduceOnly` 或相近字段
- `leverage`: 合约场景可能需要单独设置或作为下单参数

### 说明

Bitget 合约通常需要 `marginCoin`、`productType`、`tradeSide`（开/平）。适配器需根据 `reduce_only` 与仓位方向推导 `tradeSide`。

## cancel_order(order_id) / cancel_all(symbol?)

### 目标标准字段

- 取消结果返回 `ok` 与 `error`，并尽量包含 `order_id`

### Bitget 映射建议

- 单撤：使用 `orderId` 或 `clientOid` 映射
- 批量撤单：若仅支持按 `symbol` 撤单，需在参数缺失时提示

## get_open_orders(symbol?)

### 目标标准字段

- `order_id`, `symbol`, `side`, `type`, `qty`, `price`, `status`, `filled_qty`

### Bitget 映射建议

- 订单量字段：`size`/`quantity`
- 已成交量：`filledQty`/`dealSize`
- 状态统一到：`open`/`filled`/`canceled`/`rejected`/`partial`

## get_fills(symbol?, since?)

### 目标标准字段

- `fill_id`, `order_id`, `symbol`, `side`, `qty`, `price`, `fee`, `timestamp`

### Bitget 映射建议

- `timestamp` 转换为毫秒时间戳
- `fee` 使用手续费金额字段（如 `fee`/`fees`），若只有费率则计算为 `qty * price * rate`

## 错误码与限频

- 统一错误结构：`{ code, message, retryable }`
- 限频错误标记 `retryable = true`，并做指数退避
- 若返回体中包含 `code`/`msg` 字段，优先填入统一错误结构

## 适配器实现要点

- 所有字段转换后输出统一结构，不向策略层暴露 Bitget 特有字段
- 对于缺失字段，输出 `null` 并在 `error` 中说明原因
- 订单时间、成交时间统一转毫秒时间戳

## 公共与账户接口（文档2补充）

- 获取服务器时间：`GET /api/v2/public/time`
  - 限速：20 次/1s (IP)
  - 说明：返回 Unix 毫秒时间戳
- 获取交易手续费（单交易对）：`GET /api/v2/common/trade-rate`
  - 限速：10 次/1s (UID)
  - 说明：获取指定交易对费率
- 获取业务线全部交易对手续费：`GET /api/v2/common/all-trade-rate`
  - 限速：10 次/1s (UID)
  - 说明：获取业务线所有交易对费率
- 资金账户资产：`GET /api/v2/account/funding-assets`
  - 限速：5 次/1s (UID)
  - 说明：资金账户，禁止子账户请求
- 策略账户资产：`GET /api/v2/account/bot-assets`
  - 限速：5 次/1s (UID)
  - 说明：策略账户资产查询
- 资产概览：`GET /api/v2/account/all-account-balance`
  - 限速：1 次/1s (UID)
  - 说明：账户总资产概览

## 合约 REST API（常用端点摘录）

以下为你提供的官方文档内容整理，供实现/对照适配器时快速查阅。参数与字段以 Bitget 官方为准。

### 市场行情

- 获取全部交易对行情：`GET /api/v2/mix/market/tickers`
  - 限速：20 次/1s (IP)
  - 必填：`productType`（`USDT-FUTURES`/`COIN-FUTURES`/`USDC-FUTURES`）
  - 主要字段：`symbol`, `lastPr`, `askPr`, `bidPr`, `high24h`, `low24h`, `ts`, `change24h`, `baseVolume`, `quoteVolume`, `markPrice`, `fundingRate`
- 获取单个交易对行情：`GET /api/v2/mix/market/ticker`
  - 限速：20 次/1s (IP)
  - 必填：`symbol`, `productType`
  - 字段同 `tickers`
- 最近成交明细（近 100 条）：`GET /api/v2/mix/market/fills`
  - 限速：20 次/1s (IP)
  - 必填：`symbol`, `productType`；可选 `limit` (<=100)
  - 主要字段：`tradeId`, `price`, `size`, `side`, `ts`
- 成交明细历史（近 90 天）：`GET /api/v2/mix/market/fills-history`
  - 限速：10 次/1s (IP)
  - 必填：`symbol`, `productType`；可选 `startTime`, `endTime`, `limit`, `idLessThan`
  - 约束：`startTime`/`endTime` 间隔 <= 7 天
- K 线：`GET /api/v2/mix/market/candles`
  - 限速：20 次/1s (IP)，默认 100 条，最大 1000
  - 必填：`symbol`, `productType`, `granularity`
  - 可选：`startTime`, `endTime`, `kLineType`, `limit`
- 历史 K 线：`GET /api/v2/mix/market/history-candles`
  - 限速：20 次/1s (IP)，最大 200
  - 必填：`symbol`, `productType`, `granularity`
  - 可选：`startTime`, `endTime`, `limit`
- 指数 K 线历史：`GET /api/v2/mix/market/history-index-candles`
  - 限速：20 次/1s (IP)，最大 200
  - 必填：`symbol`, `productType`, `granularity`
  - 可选：`startTime`, `endTime`, `limit`
- 平台总持仓量：`GET /api/v2/mix/market/open-interest`
  - 限速：20 次/1s (IP)
  - 必填：`symbol`, `productType`
- 合约信息：`GET /api/v2/mix/market/contracts`
  - 限速：20 次/1s (IP)
  - 必填：`productType`；可选 `symbol`
  - 用于精度、最小下单量、费率、杠杆范围等映射

- 预估强平价：`GET /api/v2/mix/account/liq-price`

- 可开数量（不计委托）：`GET /api/v2/mix/account/open-count`
